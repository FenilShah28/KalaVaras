# Directives

This directory contains SOPs (Standard Operating Procedures) written in Markdown.

Each directive defines:
- **Goal**: What the directive accomplishes
- **Inputs**: What data/context is needed
- **Tools/Scripts**: Which execution scripts to use
- **Outputs**: What the directive produces
- **Edge Cases**: Known pitfalls and how to handle them

## Naming Convention

Use lowercase with underscores: `scrape_website.md`, `generate_report.md`, `process_data.md`

## Living Documents

Directives are updated as the system learns. When an execution script reveals new constraints, edge cases, or better approaches — the corresponding directive gets updated.
